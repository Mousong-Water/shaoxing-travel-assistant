# Grill With Docs

> 参考: mattpocock/skills — 改编为 Python 项目

## 触发条件

提出新架构方案、重大改动或新模块设计时。

## 质问清单

### 架构一致性
- [ ] 新方案是否遵循三层架构 (数据层/智能体层/前端层)？
- [ ] 是否引入了新的依赖链或循环导入？
- [ ] 是否复用了已有的工具类？

### 数据流
- [ ] 新模块的输入从哪来？输出到哪去？
- [ ] 是否需要更新 `run_crawler_center.py` 的数据流？
- [ ] CSV/SQLite 导出是否需要适配新字段？

### 竞赛约束
- [ ] 是否引入了需要登录的平台？
- [ ] 是否依赖付费 API？
- [ ] 数据是否可溯源/可验证？
- [ ] 是否避免了生成假数据？

## 文档更新清单

- [ ] `CLAUDE.md` — 添加新模块说明
- [ ] `PROJECT_JOURNAL.md` — 记录决策和理由
- [ ] `PROJECT_OVERVIEW.md` — 更新进度条和架构图
- [ ] `docs/adr/ADR-{序号}.md` — 如方案有争议或重大取舍

## ADR 模板

```markdown
# ADR-{序号}: {标题}
日期: YYYY-MM-DD
状态: 提议/已采纳/已废弃

## 背景 (为什么要做这个决策？)
## 决策 (选择了什么方案？)
## 备选方案 (考虑过哪些替代方案？)
## 后果 (这个决策会带来什么影响？)
```
