# Diagnose

参考: mattpocock/skills - diagnose (改编为 Python 爬虫项目)

## 核心规则

**先建立可复现的反馈回路，再形成假设。**

不要猜测原因。用数据说话。

## 六步诊断循环

### 1. Reproduce (复现)

```
- 最小复现命令是什么？
- 是否每次必现还是偶发？
- 是否与特定输入/参数有关？
```

### 2. Minimise (缩小范围)

```
- 能不能用 1 个景点复现？
- 能不能只跑出问题的爬虫？
- 去掉网络因素 (用静态数据能否复现)？
- 去掉并发因素 (单线程能否复现)？
```

### 3. Hypothesise (假设)

基于观察而非直觉:

```
- 对比正常数据 vs 异常数据的差异
- 检查日志中的 ERROR/WARNING
- 检查上游数据是否有变化 (网站改版？JSON格式错误？)
- 列出 2-3 个可能的根因
```

### 4. Instrument (插桩)

```
- 在关键位置加 print/logging
- 打印中间变量的类型和值
- 用 try/except 捕获具体异常类型
- 保存中间结果到临时文件便于检查
```

### 5. Fix (修复)

```
- 修改最小代码量
- 考虑边界情况 (空值/超时/格式变化)
- 添加防御性检查
```

### 6. Regression-test (回归)

```
- 原复现步骤不再触发
- 正常数据仍然正常处理
- 如果有 CI/CD，添加对应的测试用例
```

## 常见问题诊断工具箱

| 症状 | 可能原因 | 检查方法 |
|------|---------|---------|
| JSONDecodeError | 文件末尾多余`}` | `tail -1 *.json` |
| ModuleNotFoundError | 缺少 pip 包 | `pip list | grep xxx` |
| ConnectionReset | 网络/WAF | 用 curl 测试URL可达性 |
| 数据量骤降 | 网站改版 | 保存 HTML debug 文件对比 |
| 合并结果太少 | 阈值过严 | 打印相似度矩阵 |
| 内存溢出 | 数据量过大 | 分批处理 |

## 输出

修复后更新 `PROJECT_JOURNAL.md` 记录:
- 问题现象
- 根因分析
- 修复方法
- 如何预防复发
