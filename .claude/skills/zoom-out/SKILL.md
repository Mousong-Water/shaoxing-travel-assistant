# Zoom Out

参考: mattpocock/skills - zoom-out (改编)

## 触发条件

- 新组员需要快速了解项目
- 大型改动前理解影响范围
- 定期项目审查

## 输出格式

### 1. 模块全景图

```
{项目名}
├── {层级1}
│   ├── {模块A} — {一句话职责}
│   │   ├── 被哪些模块调用
│   │   └── 调用哪些模块
│   └── {模块B}
├── {层级2}
...
```

### 2. 数据流向

```
{数据源} → {处理模块} → {存储} → {消费模块}
```

### 3. 关键依赖

| 模块 | 依赖 | 被依赖 |
|------|------|--------|
| ... | ... | ... |

### 4. 热点标注

- 🔴 频繁变动的模块
- 🟢 稳定的基础模块
- 🟡 待开发/不完整的模块
- ⚠️ 有已知问题的模块

### 5. 新增模块指南

- 在哪一层新建文件
- 需要注册到哪些入口
- 需要更新哪些配置文件

## 约束

- 只描述"是什么"和"怎么关联"，不讨论"怎么做"
- 使用项目已建立的术语 (参考 `CLAUDE.md` 和 `PROJECT_OVERVIEW.md`)
- 标注任何不熟悉的模块 `[待深入了解]`
